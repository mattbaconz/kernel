# Code Of Conduct

Kernel contributors are expected to keep collaboration direct, respectful, and focused on the work.

## Expected Behavior

- Be clear and factual.
- Assume good intent while verifying claims with evidence.
- Discuss technical tradeoffs without personal attacks.
- Respect private security reports and user-authored files.
- Keep reviews focused on correctness, maintainability, risk, and evidence.

## Unacceptable Behavior

- Harassment, threats, or discriminatory language.
- Publishing private information without permission.
- Ignoring security handling expectations.
- Deliberately bypassing repository safety policies.

## Enforcement

Repository maintainers may remove comments, close issues, block contributors, or restrict access when behavior disrupts safe and productive collaboration.

Report conduct concerns privately to the repository owner.
